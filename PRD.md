## 1. 项目背景与目标

### 1.1 业务背景

患者的不同检查报告（如血常规、尿常规、生化、肝功能等）需要在系统中以结构化方式录入与展示。每一类检查报告包含若干**检查项**（例如白细胞计数、血红蛋白浓度等），不同检查项有：

- 不同的**展示形态**（数值、文本、下拉、单位标签等）
- 不同的**业务规则**：
  - **计算**：如平均红细胞体积 = 红细胞压积 / 红细胞计数
  - **阈值判断**：如血糖 > 7.0 mmol/L 标记为「偏高」
  - **区间判定**：如 4.0 ≤ 白细胞 ≤ 10.0 为「正常」
  - **联动**：B 值的合理范围受 A 值影响
  - **条件显示**：某些字段仅在特定条件下出现

当前的实现是**硬编码**在每个报告页面里，新增/调整一种检查报告就要改前端代码 + 后端 schema，开发成本高、维护困难。

### 1.2 项目目标

构建一套**配置化**的动态表单系统，让运营/医务人员通过可视化设计器配置每个检查报告的字段与规则，保存为 JSON 模板，前端**渲染器**消费模板即可渲染页面，**无需发版**即可新增报告类型。

### 1.3 原型阶段范围

本 PRD 聚焦于**原型阶段**，交付物为可运行的 Vue 3 + Vite + Antd Vue3 单页应用，覆盖以下核心链路：

1. 模板列表（CRUD，本地存储）
2. 表单设计器（拖拽 + 规则编辑）
3. 表单渲染器（按 JSON 渲染 + 规则实时执行）
4. JSON Schema 定义 + 导出/导入

**不在原型范围**：权限、多人协作、版本管理、后端持久化（用 localStorage 模拟）、真实医学规则库。

## 2. 名词定义

| 名词       | 含义                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 模板       | 一份检查报告的结构化 JSON 描述，对应一种报告类型（如「血常规」）     |
| 组件       | 模板中的最小 UI 单元，对应一个检查项（如白细胞计数）                 |
| 组件类型   | 组件的展示形态（输入、选择、日期等），即 antd vue3 控件的一种       |
| 规则       | 附加在组件上的一段可执行逻辑，影响该组件或其他组件的值/样式/显隐    |
| 规则组     | 一个组件上的多条规则的集合                                          |
| 表达式     | 引用其他组件值或常量进行计算的 DSL 片段                              |
| 画布       | 设计器中间区域，用于放置与调整组件                                  |
| 渲染器     | 接收 JSON 模板、动态渲染出表单页面的运行时组件                       |
| 规则引擎   | 监听值变化、按规则求值并应用副作用的核心模块                        |

│  │   ├─ FieldRenderer  按 type 渲染 antd 组件         │  │
│  │   ├─ RuleEngine     值变化时执行规则               │  │

└──────────────────────────────────────────────────────────┘

**关键架构决策**：

| 决策点              | 选型                                                | 理由                                         |
| ------------------- | --------------------------------------------------- | -------------------------------------------- |
| 目标用户            | 非技术人员（运营/医务人员）                         | 编辑器零代码化为最高优先级                   |
| 规则复杂度分级      | L1 零代码（阈值/显隐/必填） / L2 模板化（求和/平均） / L3 表达式（自定义） | 90% 场景不暴露表达式，必要时才让用户写       |
| 规则表达式          | JSON 结构化参数 + 简易表达式（`a + b / 2`）         | 安全性、可视化编辑、零外部依赖               |
| 后端模拟            | localStorage（封装为 `templateApi`）                | 原型阶段最快，预留切换为 fetch 的能力        |
| 规则编辑器交互      | 表单+下拉为主，表达式编辑器带「插入字段」与「运算符」按钮组 | 用户无需记忆 `$.components.xxx.value` 语法   |
| 新增规则入口        | 向导式 4 步弹窗（类型→触发源→参数→副作用）         | 降低首次配置的心智负担                       |
| 拖拽库              | `vuedraggable@next`（基于 Sortable.js）             | Vue 3 生态最稳、文档完善、画布与组件库通用   |
| JSON 路径引用       | 自研轻量路径解析 `$.components.xxx.value`           | 比 JSONPath 简单够用，无需引入第三方         |

## 5. 数据模型与 JSON Schema

### 5.1 模板整体结构（FormTemplate）

```json
{
  "version": "1.0.0",
  "id": "tmpl_blood_routine",
  "name": "血常规检查报告",
  "category": "血液检查",
  "description": "用于血常规检查报告录入",
  "layout": {
    "type": "form",
    "columns": 2,
    "labelPosition": "right",
    "labelWidth": 120
  },
  "components": [
    /* ComponentDef[]，见 5.2 */
  ]
}
```

### 5.2 组件定义（ComponentDef）

```json
{
  "id": "comp_wbc",
  "type": "InputNumber",
  "field": "wbc",
  "label": "白细胞计数",
  "unit": "10^9/L",
  "defaultValue": null,
  "required": true,
  "props": {
    "min": 0,
    "max": 100,
    "precision": 2,
    "placeholder": "请输入白细胞计数"
  },
  "options": [
    /* 仅 Select/Radio/Checkbox 需要 */
    { "label": "男", "value": "male" },
    { "label": "女", "value": "female" }
  ],
  "rules": [
    /* RuleDef[]，见 5.3 */
  ]
}
```

**字段说明**：

| 字段          | 类型                | 说明                                                              |
| ------------- | ------------------- | ----------------------------------------------------------------- |
| `id`          | string              | 模板内唯一 ID，规则表达式通过此 ID 引用                          |
| `type`        | enum                | 组件类型（见 5.4 组件类型表）                                     |
| `field`       | string              | 数据字段名，提交数据时的 key                                      |
| `label`       | string              | 表单 label                                                        |
| `unit`        | string?             | 单位展示（医学报告常用）                                          |
| `defaultValue`| any                 | 默认值                                                            |
| `required`    | boolean             | 是否必填（同步生成基础校验规则）                                  |
| `props`       | object              | 透传给 antd 组件的属性                                            |
| `options`     | array?              | 仅选择类组件需要                                                  |
| `rules`       | RuleDef[]           | 附加在该组件上的规则组                                            |

### 5.3 规则定义（RuleDef）

```json
{
  "id": "rule_wbc_threshold",
  "type": "threshold",
  "name": "白细胞异常分级",
  "enabled": true,
  "trigger": "onChange",
  "params": { /* 不同 type 的参数，见 5.5 */ }
}
```

通用字段：

| 字段     | 类型                       | 说明                                              |
| -------- | -------------------------- | ------------------------------------------------- |
| `id`     | string                     | 规则 ID                                           |
| `type`   | enum                       | 规则类型（见 5.5 规则类型表）                     |
| `name`   | string                     | 规则名称（设计器展示用）                          |
| `enabled`| boolean                    | 是否启用                                          |
| `trigger`| enum: `onChange`/`onBlur`  | 触发时机                                          |
| `params` | object                     | 规则类型相关参数                                  |

### 5.4 组件类型表

| type             | 对应 antd 组件           | 用途                | 关键 props                |
| ---------------- | ------------------------ | ------------------- | ------------------------- |
| `Input`          | `a-input`                | 单行文本            | maxLength                 |
| `Textarea`       | `a-textarea`             | 多行文本            | rows, maxLength           |
| `InputNumber`    | `a-input-number`         | 数值                | min, max, precision, step |
| `Select`         | `a-select`               | 下拉单选            | mode, options             |
| `RadioGroup`     | `a-radio-group`          | 单选                | options                   |
| `CheckboxGroup`  | `a-checkbox-group`       | 多选                | options                   |
| `DatePicker`     | `a-date-picker`          | 日期                | format, valueFormat      |
| `TimePicker`     | `a-time-picker`          | 时间                | format, valueFormat      |
| `RangePicker`    | `a-range-picker`         | 日期范围            | format                    |
| `Switch`         | `a-switch`               | 开关                | -                         |
| `Slider`         | `a-slider`               | 滑块                | min, max, step            |
| `DisplayText`    | 自定义只读文本           | 展示计算结果        | prefix, suffix            |

> 原型阶段至少实现 **8 种**：Input、InputNumber、Select、RadioGroup、CheckboxGroup、DatePicker、Switch、DisplayText。

### 5.5 规则类型表

#### 5.5.1 `calculation` 计算规则

引用其他组件的值进行数学运算，结果写入当前组件。**面向非技术用户的编辑方式**：

**L2 模板化（推荐，覆盖 80% 场景）**：从预设模板中选择，自动生成表达式。

| 模板              | 自动表达式                                                |
| ----------------- | --------------------------------------------------------- |
| `sum`             | `$.components.A.value + $.components.B.value + ...`       |
| `average`         | `(A + B + C) / N`                                         |
| `max`             | `Math.max(A, B, C)`                                       |
| `min`             | `Math.min(A, B, C)`                                       |
| `percent`         | `(A / B) * 100`                                           |
| `ratio`           | `(A / B) * 100%`                                          |
| `subtract`        | `A - B`                                                   |
| `custom`          | 暴露表达式编辑器（见下）                                  |

模板对应的 JSON：

```json
{
  "id": "rule_mcv",
  "type": "calculation",
  "name": "平均红细胞体积",
  "enabled": true,
  "trigger": "onChange",
  "params": {
    "template": "custom",
    "expression": "($.components.hct.value / $.components.rbc.value)",
    "precision": 2
  }
}
```

- `template`：所选预设模板（`sum`/`average`/`max`/`min`/`percent`/`ratio`/`subtract`/`custom`）
- `expression`：当 `template=custom` 时由用户编写，其他情况由模板自动生成、可二次修改
- `precision`：可选，控制小数位

#### 5.5.2 `threshold` 阈值规则（多区间分级）—— **L1 零代码**

**面向非技术用户的编辑方式**：纯表单配置，无需写任何表达式。

```json
{
  "id": "rule_wbc_threshold",
  "type": "threshold",
  "name": "白细胞分级",
  "enabled": true,
  "trigger": "onChange",
  "params": {
    "ranges": [
      { "min": null,  "max": 4,   "label": "偏低", "level": "low",     "color": "#52c41a" },
      { "min": 4,     "max": 10,  "label": "正常", "level": "normal",  "color": "#1890ff" },
      { "min": 10,    "max": null,"label": "偏高", "level": "high",    "color": "#f5222d" }
    ],
    "actions": [
      { "type": "setStyle", "target": "$.self",     "style": { "color": "$level.color" } },
      { "type": "setText",  "target": "$.self.suffix", "value": "$level.label" }
    ]
  }
}
```

- `null` 表示无下限/上限
- `level` 用于联动样式与文案
- `actions` 是副作用列表（详见 5.6）
- **UI 编辑**：用「+ 添加区间」按钮，每行表单填写 min/max/label/颜色，**用户完全不接触 JSON**

```json
{
  "id": "rule_wbc_threshold",
  "type": "threshold",
  "name": "白细胞分级",
  "enabled": true,
  "trigger": "onChange",
  "params": {
    "ranges": [
      { "min": null,  "max": 4,   "label": "偏低", "level": "low",     "color": "#52c41a" },
      { "min": 4,     "max": 10,  "label": "正常", "level": "normal",  "color": "#1890ff" },
      { "min": 10,    "max": null,"label": "偏高", "level": "high",    "color": "#f5222d" }
    ],
    "actions": [
      { "type": "setStyle", "target": "$.self",     "style": { "color": "$level.color" } },
      { "type": "setText",  "target": "$.self.suffix", "value": "$level.label" }
    ]
  }
}
```

- `null` 表示无下限/上限
- `level` 用于联动样式与文案
- `actions` 是副作用列表（详见 5.6）

#### 5.5.3 `comparison` 比较规则 —— **L1 零代码**

**面向非技术用户的编辑方式**：三段式表单（下拉选字段 + 下拉选操作符 + 输入框填值），无需写表达式。

```json
{
  "id": "rule_compare",
  "type": "comparison",
  "name": "血糖高于阈值",
  "enabled": true,
  "trigger": "onChange",
  "params": {
    "operator": ">",
    "left":  "$.self.value",
    "right": { "type": "constant", "value": 7.0 },
    "actions": [
      { "type": "addClass", "target": "$.self", "value": "is-abnormal" }
    ]
  }
}
```

- `operator`: `>` `>=` `<` `<=` `==` `!=` `between`
- `left`/`right`: 值来源，`{ "type": "constant", "value": X }` 或 `$.components.xxx.value`
- **UI 编辑**：左字段下拉 → 操作符下拉 → 右值（常量输入框 / 字段下拉二选一），**全部下拉选择**

与另一个组件的值比较，可触发样式变化或显隐。

```json
{
  "id": "rule_compare",
  "type": "comparison",
  "name": "血糖高于阈值",
  "enabled": true,
  "trigger": "onChange",
  "params": {
    "operator": ">",
    "left":  "$.self.value",
    "right": { "type": "constant", "value": 7.0 },
    "actions": [
      { "type": "addClass", "target": "$.self", "value": "is-abnormal" }
    ]
  }
}
```

- `operator`: `>` `>=` `<` `<=` `==` `!=` `between`
- `left`/`right`: 值来源，`{ "type": "constant", "value": X }` 或 `$.components.xxx.value`

#### 5.5.5 `validation` 校验规则 —— **L1 零代码**

**面向非技术用户的编辑方式**：勾选式表单 + 输入框填阈值，**完全无代码**。

```json
{
  "id": "rule_validate",
  "type": "validation",
  "name": "白细胞必填且为正数",
  "enabled": true,
  "trigger": "onBlur",
  "params": {
    "validators": [
      { "type": "required",  "message": "白细胞计数不能为空" },
      { "type": "min",       "value": 0,         "message": "不能小于 0" },
      { "type": "max",       "value": 100,       "message": "不能大于 100" },
      { "type": "regex",     "pattern": "^\\d+(\\.\\d+)?$", "message": "请输入数字" }
    ]
  }
}
```

`required` 字段会自动生成 `{ type: "required" }` 一条规则，无需手填。**UI 编辑**：勾选「必填」「数值范围」「正则」+ 输入数字 + 输入错误提示文案，**用户不接触正则表达式**（若选了正则，提供常见模式按钮如「整数」「手机号」「身份证」）。

### 5.6 副作用（Action）

| type        | 参数                                  | 作用                       |
| ----------- | ------------------------------------- | -------------------------- |
| `setValue`  | target, value                         | 设置目标组件值             |
| `setStyle`  | target, style                         | 设置内联样式（支持 `$level.color` 占位） |
| `addClass`  | target, value                         | 添加 class                  |
| `removeClass` | target, value                       | 移除 class                  |
| `setText`   | target, value                         | 设置文本（多用于 suffix/单位旁） |
| `show`      | target                                | 显示目标组件               |
| `hidden`    | target                                | 隐藏目标组件               |
| `setRequired` | target, value                       | 设置是否必填               |

target 支持：

- `$.self`：当前组件
- `$.self.suffix`：当前组件的 suffix 文本节点
- `$.components.{id}`：其他组件

## 12. 非技术用户友好性设计（核心）

> 目标用户**不懂开发**——可能是医生、护士、运营人员。本节是整个 PRD 的最高优先级约束，所有功能设计都必须先回答一个问题：「一个不懂代码的人能不能独立完成这个操作？」

### 12.1 设计原则

| 原则                        | 说明                                                                                              |
| --------------------------- | ------------------------------------------------------------------------------------------------- |
| **零代码优先**              | 90% 的规则通过下拉选择 + 表单填写完成，绝不让用户写表达式                                          |
| **自然语言反馈**            | 所有规则配置完成后，用一句中文自然语言描述给用户看（如「当白细胞 < 4 时，显示『偏低』并标红」）    |
| **所见即所得**              | 表达式编辑器实时预览计算结果，让用户立刻知道表达式是不是写对了                                      |
| **向导代替表单**            | 新增规则用 4 步向导，不要一上来就甩出一堆字段                                                     |
| **示例驱动**                | 每个规则类型在向导 Step 1 旁边展示「示例效果」（静态截图或动图）                                   |
| **可逆操作**                | 所有修改都有撤销/重做；向导可随时返回上一步                                                       |
| **危险操作明示**            | 删除规则、删除模板等用 antd `a-popconfirm` 二次确认                                               |

### 12.2 规则复杂度分级（核心模型）

| 级别   | 含义                            | 典型规则                                                  | 编辑方式                                                            |
| ------ | ------------------------------- | --------------------------------------------------------- | ------------------------------------------------------------------- |
| **L1** | 零代码（占 60%）                | 必填、数值范围、阈值分级、条件显隐                        | 纯表单 + 下拉 + 勾选开关                                            |
| **L2** | 预设模板（占 30%）              | 求和、求平均、最大、最小、百分比、占比、差值              | 选预设模板 → 自动填好表达式 → 用户确认                              |
| **L3** | 自定义表达式（占 10%）          | 任意自定义计算（`a + b/2`、`a*0.85+5`）                  | 表达式编辑器，但带「插入字段」按钮和「运算符」按钮组                |

**所有规则编辑器都必须从 L1 开始，让用户在向导里显式选择「自定义」才会跳到 L3。**

### 12.3 新增规则的 4 步向导

```
┌──────────────────────────────────────────────────────────┐
│  新增规则                                       [×]        │
├──────────────────────────────────────────────────────────┤
│  ① 选择类型        ② 谁触发         ③ 配置参数    ④ 副作用 │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  请选择要配置的规则类型：                                  │
│                                                          │
│  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐           │
│  │ 阈值    │  │ 条件   │  │ 求和   │  │ 求平均 │           │
│  │ 分级   │  │ 显隐   │  │        │  │        │           │
│  │ [图]   │  │ [图]   │  │ [图]   │  │ [图]   │           │
│  │ 「值    │  │「A为  │  │「A+B+C │  │「(A+B+C│           │
│  │ < 4    │  │ X 时   │  │ 」     │  │)/3    │           │
│  │ 偏低」 │  │ 显示B」│  │        │  │ 」     │           │
│  └────────┘  └────────┘  └────────┘  └────────┘           │
│                                                          │

└──────────────────────────────────────────────────────────┘
```

| 步骤      | 内容                                                                                            |
| --------- | ----------------------------------------------------------------------------------------------- |
| Step 1    | 规则类型选择（图标 + 一句话说明 + 缩略图示例），大卡片点击选择                                  |
| Step 2    | 谁的值变化时触发？——下拉选择组件（如「白细胞计数」）                                            |
| Step 3    | 配置参数——按规则类型动态展示对应表单（L1 纯表单 / L2 模板选择 / L3 表达式编辑器）             |
| Step 4    | 配置副作用（可选）——勾选「异常时高亮」「异常时加文字标注」「触发其他字段重算」                  |

向导最后一步展示「**规则预览**」：

> 「**白细胞计数** < 4 时，将该字段标记为**『偏低』**并**标红**显示」

### 12.4 L3 表达式编辑器增强

```
┌──────────────────────────────────────────────────────────┐
│  [插入字段 ▼]  [+]  [-]  [×]  [÷]  [(]  [)]   ✓ 语法正确  │
├──────────────────────────────────────────────────────────┤
│  ($[●红细胞压积] + $[●血红蛋白]) / 2                     │
├──────────────────────────────────────────────────────────┤
│  实时预览：23.45                                          │
│  说明：使用 [插入字段] 按钮引用其他组件的值                  │
└──────────────────────────────────────────────────────────┘
```

**关键能力**：

| 能力                 | 说明                                                                                                |
| -------------------- | --------------------------------------------------------------------------------------------------- |
| **插入字段按钮**     | 下拉选择其他组件，选中后插入 `$.components.{id}.value` 文本片段，光标定位到字段名上                  |
| **运算符按钮组**     | `+ - × ÷ ( )` 一键插入，避免用户切换输入法                                                          |
| **路径高亮**         | `$.components.xxx.value` 整段用蓝色 token 高亮，与用户输入的运算符/数字视觉区分                    |
| **实时预览**         | 输入过程中实时计算结果（用当前值或 mock 值）                                                        |
| **语法校验提示**     | 括号不匹配、引用了不存在的组件时给红色提示                                                          |
| **自然语言翻译**     | 下方自动显示「白细胞 / 红细胞 + 血红蛋白 / 2」的文字描述                                            |

### 12.5 规则的"自然语言描述"

每条规则配置完成后，在规则列表里展示一句中文描述：

| 规则类型         | 描述模板                                                                  |
| ---------------- | ------------------------------------------------------------------------- |
| `threshold`      | 「当 **白细胞计数** 在 **[4, 10)** 区间时，显示 **『正常』** 文字并标记为蓝色」 |
| `calculation`    | 「**平均红细胞体积** 自动等于 **红细胞压积 ÷ 红细胞计数**（保留 2 位小数）」 |
| `comparison`     | 「当 **血糖** **> 7.0** 时，将该字段标记为偏高并加红框」                   |
| `conditional`    | 「当 **性别** 为 **女** 时，显示 **『妊娠相关胆红素』** 字段」              |
| `validation`     | 「**白细胞计数** 不能为空，且必须 ≥ 0」                                    |

描述由规则参数自动生成（规则引擎提供一个 `describe(rule, components)` 函数），用户配置完规则后立即看到这句话，确认"是不是这个意思"。

### 14.3 关键技术决策（实施中确定）

| 决策点 | 选型 | 原因 |
| --- | --- | --- |
| 拖拽库 | **SortableJS 直接调用** | vuedraggable 4 在 Vue 3.5 + antd vue3 下有兼容问题 |
| 字段宽度 | **`columns: number`（12 进制，参考 vueform）** | 不是 colSpan: 24，与 vueform 设计哲学一致 |
| 字段布局 | `flex-wrap: wrap` + `flex-basis: calc(N/12 * 100% - gap)` | SortableJS 只支持一维列表，grid 24 列不兼容 |
| 设计器与渲染器一致性 | **用同一套 columns→width 公式** | WYSIWYG（设计器所见即所得） |
| 表达式求值 | 自研轻量递归下降解析器 | 需求简单，无需第三方数学库 |
| 默认 columns | 短字段 6（半行），长字段 12（整行） | 拖入新组件给合理默认值 |

### 14.4 已修复的关键 Bug（避免重复踩坑）

1. **画布空状态无法接收 drop** → canvas-list 容器始终渲染，空状态内部用 `v-if`
2. **CSS Grid 24 列 + SortableJS 不兼容** → 改用 flex-wrap
3. **空状态 div 被 SortableJS 当作 sortable item** → 加 `draggable: '.canvas-item'` + `filter: '.empty-state'`
4. **导出空模板** → 新增 `templateApi.exportObject(template)` 直接用内存对象
5. **左侧组件库无法滚动** → flex container + `min-height: 0` + 内容区 `overflow-y: auto`
6. **vuedraggable 4 + Vue 3.5 兼容问题** → 改用 SortableJS

### 14.7 关键交互细节

- **拖拽从空状态开始**：画布空时显示蓝色虚线 dropzone，拖入即落位
- **字段宽度按钮**：每个字段底部 4 个按钮（▭12/◫6/◩4/◪3）一键切换
- **宽度按钮在右侧属性面板也有**：a-slider 3-12 任意值
- **自然语言描述**：每条规则下方显示中文描述（例：「当 白细胞计数 在 4~10 区间时，显示 正常 文字」）
- **规则引擎实时执行**：渲染器中输入数值时，threshold 规则的 setText 副作用立即把"偏低/正常/偏高"附加到字段后缀

### 14.8 用户偏好的硬约束（开发时遵守）

1. **WYSIWYG 优先**：设计器看到什么，渲染器就显示什么，不接受"两套视觉"
2. **仿 vueform 视觉**：左侧类型图标 + 中间 label + 右侧操作；去 emoji 化用 SVG/字母
3. **零代码化**：能不让用户写表达式就不让；L1/L2/L3 分级只在自定义时才暴露 L3
4. **所见即所得的字段宽度**：用 columns 字段，不用 colSpan；设计器和渲染器用同一公式
